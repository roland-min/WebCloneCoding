버전관리

2024-12-21 10:00:00

  • Git은 분산 버전 관리 시스템(DVCS)입니다.
  • 파일의 변경 사항을 기록하여 이후 특정 버전으로 되돌릴 수 있습니다.
  • 효율적인 협업 환경을 제공하며, 코드 변경 충돌을 최소화합니다.
  • 대규모 프로젝트에서 안정적인 파일 관리 및 추적을 지원합니다.

버전 관리란?

  • 버전 관리는 파일의 변화를 시간에 따라 기록하고, 과거 버전으로 되돌릴 수 있는 기능입니다.
  • 예: 프로젝트 초기 상태를 저장한 후, 각 수정 단계마다 기록을 남기는 방식.
  • Git은 수동 관리 방식보다 훨씬 효율적이며, 작업 히스토리를 체계적으로 관리합니다.

Git의 주요 명령어

  • git init: 새로운 Git 저장소를 초기화합니다.
  • git add: 변경된 파일을 스테이지 영역에 추가합니다.
  • git commit: 스테이지 영역의 변경 사항을 로컬 저장소에 저장합니다.
  • git status: 현재 작업 디렉토리와 스테이지 상태를 확인합니다.
  • git log: 커밋 히스토리를 보여줍니다.
  • git clone: 원격 저장소를 복제하여 로컬에 저장합니다.
  • git pull: 원격 저장소의 최신 변경 사항을 가져옵니다.
  • git push: 로컬 변경 사항을 원격 저장소에 반영합니다.

수동 버전 관리의 한계

  • 파일 이름을 변경하여 다른 버전을 관리하는 방식은 비효율적입니다.
  • 수백 개의 파일이 있는 대규모 프로젝트에서는 충돌 위험이 높습니다.
  • Git과 같은 버전 관리 시스템은 이러한 문제를 자동으로 해결합니다.
